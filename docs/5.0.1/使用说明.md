# Keel-MySQL 5.0.1 使用说明

本文档对应仓库中的 **5.0.1** 开发线（`gradle.properties` 中 `version=5.0.1-SNAPSHOT`）。正式发布后，请将依赖坐标中的版本号替换为实际发布的
`5.0.1` 工件版本。

**分主题文档
**（按模块与场景拆分）：见同目录 [README.md](index.md)（含导航表与场景速查）；常见串联示例见 [场景实践汇编.md](./场景实践汇编.md)。

## 概述

**Keel-MySQL** 是 [Keel](https://github.com/sinri/keel) 框架的 MySQL 集成库，在 [Eclipse Vert.x
`vertx-mysql-client`](https://vertx.io/docs/vertx-mysql-client/java/) 之上提供：

- 基于 Keel 配置树的**命名数据源**与连接池；
- **`NamedMySQLDataSource`**：借还连接、事务封装、池指标与（可选）虚拟线程取连；
- **`NamedMySQLConnection`**：通过 **`RunnableStatementFactory`** 构建 SELECT/INSERT/UPDATE/DDL、原始 SQL、模板 SQL 等；
- 结果封装（如 **`ResultMatrix`**、分页 **`PaginationResult`** 等）。

运行模型为 **Vert.x 异步 Future**，需与已初始化的 **`Vertx`** 及 Keel 的全局配置（**`ConfigElement.root()`**）配合使用。

| 项目        | 要求                                                             |
|-----------|----------------------------------------------------------------|

使用前请确保应用已按 **Keel** 文档加载配置（例如属性文件映射到 `ConfigElement` 根节点），否则 *
*`KeelMySQLDataSourceProvider.getMySQLConfiguration(...)`** 会失败。

## 配置说明

配置挂在 Keel 配置树的 **`mysql.<数据源名称>`** 节点下。常用键如下（名称区分大小写，取决于 Keel 配置实现；以下与 *
*`KeelMySQLConfiguration`** 源码一致）：

| 键                       | 说明                       | 默认或未配置时      |
|-------------------------|--------------------------|--------------|
| `host`                  | 主机                       | `127.0.0.1`  |
| `port`                  | 端口                       | `3306`       |
| `username`              | 用户                       | 无默认，需有效配置    |
| `password`              | 密码                       | 可为空          |
| `schema`                | 数据库名                     | 可选           |
| `charset`               | 字符集                      | 可选           |
| `poolMaxSize`           | 池最大连接数                   | 使用 Vert.x 默认 |
| `poolShared`            | 是否共享池（布尔，配置中为 YES/NO 形式） | `true`       |
| `poolConnectionTimeout` | 从池取连超时（**秒**）            | 未设置则不由本类写入超时 |

此外，在根配置可提供：

- **`mysql.default_data_source_name`**：默认数据源逻辑名；未配置时为 **`default`**。

连接选项中 **`useAffectedRows`** 会被设为 **`true`**（与 Vert.x **`MySQLConnectOptions`** 行为一致）。

### 配置模板（推荐，5.0.1）

**不要**使用已弃用的 **`generatePropertiesForConfig`**（会把密码明文写入字符串）。请使用 *
*`KeelMySQLConfiguration.generateSamplePropertiesForConfig(String dataSourceName)`** 生成**仅含占位符
**的样本，再手工填入真实连接信息或改为环境变量/密钥管理。

示例占位输出结构（示意）：

```properties
mysql.mydb.host=127.0.0.1
mysql.mydb.port=3306
mysql.mydb.username=<YOUR_USERNAME>
mysql.mydb.password=<YOUR_PASSWORD>
mysql.mydb.schema=<YOUR_DATABASE>
mysql.mydb.charset=utf8mb4
mysql.mydb.poolMaxSize=10
mysql.mydb.poolShared=YES
mysql.mydb.poolConnectionTimeout=30
```

将上述键合并进应用实际使用的 Keel 配置文件即可。

## 加载数据源

使用 **`KeelMySQLDataSourceProvider`**：

1. **`getMySQLConfiguration(String dataSourceName)`**  
   从根配置解析出 **`KeelMySQLConfiguration`**（配置缺失时抛 **`NotConfiguredException`**）。

2. **`load(Vertx, String dataSourceName, Function<SqlConnection, C> wrapper, ...)`**  
   构建 **`NamedMySQLDataSource<C>`**，并在首连上做**连通性探测**；超时时长来自 **`PoolOptions`** 中的连接超时设置。

3. **`loadDefault(Vertx)`** / **`loadDynamic(Vertx, name)`**  
   使用内置的 **`DynamicNamedMySQLConnection`** 包装，适合无需自定义连接类型的场景。

可选的 **`connectionSetUpFunction`**：池在 **`withConnectHandler`** 中对新连接执行自定义初始化（例如设置会话变量），完成后连接会关闭并进入池（初始化流程由
**`NamedMySQLDataSource`** 管理）。

### 自定义连接类型

继承 **`AbstractNamedMySQLConnection`**（或自实现 **`NamedMySQLConnection`**），在 **`load`** 中传入 *
*`sqlConnection -> new MyConnection(sqlConnection, dataSourceName)`**，即可在连接对象上挂载项目专用的仓储方法，同时保留 *
*`select` / `update` / `rawForPreparedQuery`** 等入口。

## 使用连接与事务

在 **`NamedMySQLDataSource`** 上：

| 方法                                                 | 说明                                                                                             |
|----------------------------------------------------|------------------------------------------------------------------------------------------------|
| **`withConnection`**                               | 借连 → 执行 **`Future`** → 自动关闭连接；失败封装为 **`KeelMySQLException`**                                   |
| **`withTransaction`**                              | 在 **`begin` / `commit` / `rollback`** 生命周期内执行；回滚语义见源码中对 **`TransactionRollbackException`** 的分支 |
| **`executeInConnection` / `executeInTransaction`** | 返回 **`ValueBox<T>`** 的变体（标记为技术预览 API）                                                          |

**`NamedMySQLConnection`** 自身也提供 **`beginTransaction` / `commitTransaction` / `rollbackTransaction`
** 等默认方法，适用于你在单连接上手工控制事务的场景。

## 虚拟线程中取连与归还（5.0.1）

若在 **虚拟线程** 中需要阻塞等待连接，可使用：

- **`fetchConnectionInVirtualThread()`**：内部通过 **`Future.await()`** 获取连接，并维护活跃计数。
- 使用完毕后必须调用 **`returnConnectionFromVirtualThread(connection)`** 归还，以正确 **`close`** 并递减计数。

**禁止**在 Vert.x **事件循环线程**上调用 **`fetchConnectionInVirtualThread()`**（会阻塞事件循环）。若未启用虚拟线程，将抛出
**`UnsupportedOperationException`**。

## 连接池与版本信息

| API                                     | 说明                                                             |
|-----------------------------------------|----------------------------------------------------------------|
| **`getCurrentPoolSize()`**              | 池内当前连接数近似值（Vert.x **`Pool#size()`**）                           |
| **`getCurrentIdleConnectionCount()`**   | 空闲连接近似值                                                        |
| **`getCurrentActiveConnectionCount()`** | 已借出未归还连接数                                                      |
| **`getFullVersion()`**                  | 首次初始化时通过 **`SELECT VERSION()`** 缓存的 MySQL 版本字符串，可能为 **`null`** |

**已弃用**：**`getCurrentInitializedConnectionCount()`** 请改用 **`getCurrentPoolSize()`**。

## 构建与执行 SQL

实现了 **`NamedMySQLConnection`** 的类型可通过 **`RunnableStatementFactory`** 上的快捷方法构建语句，例如：

- **`select(handler)`**、**`update`**、**`delete`**、**`insert`**、**`replace`**
- **`union`**、**`call`**
- DDL：**`createTable`**、**`alterTable`**、**`truncateTable`**、视图相关方法等
- **`rawForPreparedQuery(sql)`**：走预编译协议；**字符串中不应包含 `?` 占位符**（当前实现不支持绑定参数，否则易参数个数不匹配）
- **`rawForDirectQuery(sql)`**：普通查询协议
- **`templatedRead` / `templatedModify`**：基于模板路径与 **`TemplateArgumentMapping`**

也可独立 **`new SelectStatement()`** 等，再 **`attachToConnection(namedMySQLConnection.getSqlConnection())`** 后 *
*`execute()`**，与测试代码 **`Usage`** 类中的写法一致。

执行结果类型因语句而异，常见如 **`StatementExecuteResult`**、分页 **`executeForPagination`**、分类 *
*`executeForCategorizedMap`** 等，详见对应语句类与 **`result`** 包。

## 一次性查询与流式读取

在 **`KeelMySQLConfiguration`** 上：

- **`instantQuery(Vertx, sql)`**：临时创建客户端执行查询并关闭（技术预览）；需关注池共享与命名，避免资源泄漏。
- **`instantQueryForStream(Keel, sql, readWindowSize, readWindowFunction)`**：使用游标按批读取，在 *
  *`keel.asyncCallRepeatedly`** 中驱动循环，直到无更多行；适合大结果集，注意在合成链路中关闭 **`pool`**（实现已 *
  *`eventually`** 关闭相关资源）。

以上 API 仍要求调用方保证 SQL 已做合法性与注入防护。

## SQL 审计日志

**`StatementAuditorHolder`** 提供全局 **`SpecificLogger<MySQLAuditSpecificLog>`**，默认对接静默日志；可通过 *
*`reloadSqlAuditLogger(LoggerFactory)`** 挂接到实际 **`LoggerFactory`** 以输出审计事件。

## 5.0.1 相对 5.0.0 的要点

- 新增 **`generateSamplePropertiesForConfig`**；**`generatePropertiesForConfig`** 标记弃用并计划移除（安全原因）。
- **`NamedMySQLDataSource.returnConnectionFromVirtualThread`**：与 **`fetchConnectionInVirtualThread`** 配对归还连接。
- 推荐使用 **`getCurrentPoolSize()`** 替代已弃用的 **`getCurrentInitializedConnectionCount()`**。
